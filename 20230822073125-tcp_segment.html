Made up of:

1. Header

1.1. Source port

16-bits

A high numbered port chosen from a special section of ports known as ephemeral ports. Required to keep lots of outgoing connections separate.

1.2. Destination port

16-bits The port of the service the traffic is intended for.

1.3. Sequence number.

32-bits Used to keep track of where in a sequence of TCP segments this one is expected to be.

An Ethernet Frame is usually limited in size to 1,518 bytes,

1.4. Acknowledgement number.

32-bits Number of the next expected segment.

1.5. Data offset Field

4-bit Communicates how long the TCP header for this segment is.

1.6. Control Flags

6-bits Reserved for the six TCP Control Flags

1.7. TCP Window

A TCP window specifies the range of sequence numbers that might be sent before an acknowledgment is required.

1.8. Checksum

16-bits Once all of this segment has been ingested by recipient, the checksum is calculated across the entire segment and is compared with the checksum in the header to make sure that there was no data lost or corruption along the way.

1.9. Urgent Pointer

16-bits Used in conjuction with one of the TCP control flags to point out particular segments that might be more important than others.

1.10. Options Pointer

Sometimes used for more complicated flow control protocols.

1.11. Padding

Sequence of zeros to ensure that the data payload section begins at the expected location.

2. Payload

Contents of data application wants to send to each other
