#+title: Cryptographic Hashing

Cryptographic [[id:48e1ad8d-51d5-4656-b292-8f2ba311f061][Hashing]] is disticntly different from [[id:3389ae98-fd3b-4efc-88d7-e9befaae940c][encryption]] because cryptographic [[id:48e1ad8d-51d5-4656-b292-8f2ba311f061][Hash Functions]] should be =one directional=

The ideal Cryptographic [[id:48e1ad8d-51d5-4656-b292-8f2ba311f061][Hash]] Function should be =deterministic=, meaning that the *same input* value should always return the *same hash* value.
