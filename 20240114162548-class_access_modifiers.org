* Class Access Modifiers

Like ~variables~ and ~methods~, you can apply [[id:0d43fdee-5463-4f66-8b65-ae9a7b4b74e2][Access Modifiers]] to =classes=.

** _Exam Trick_

#+BEGIN_SRC java
// Bear.java
class Bird {}
class Bear {}
class Fish {}
#+END_SRC

Trying to declare a [[id:f7f87f85-1d9b-468a-8947-cd839883c9fa][Top-Level]] class with ~protected~ or ~private~ class will lead to a compiler error, though:

#+BEGIN_SRC java
// ClownFish.java
protected class ClownFish{} // DOES NOT COMPILE
// BlueTang.java
private class BlueTang {} // DOES NOT COMPILE
#+END_SRC

Does that mean a class can never be declared protected or private?
Not exactly. In [[id:77ca30c7-05f2-4219-a8b5-c552c3f1eced][7. Beyond Classes]], we present ~nested~ types and show that when you define a =class= *inside another,* it
can use any ~access modifier~.
