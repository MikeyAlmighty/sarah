* For Loop (Basic)

The *for* *loop* statement starts with the keyword ~for~, followed by a pair of parentheses enclosing the control structure of the loop.

This structure consists of (Separated by ~;~):
- =initial-action=
- =loop-continuation-condition=
- =action-after-each-iteration=

** Syntax

#+BEGIN_SRC java

for (initial-action; loop-continuation-condition; action-after-each-iteration) {
    // Loop body;
    Statement(s);
}
#+END_SRC

*** Example

#+begin_src java
for (int i = 0; i < 5; i ++) {
    System.out.println(i);
}
#+end_src

** 1. Creating an infinite loop

Although this *for* *loop* may look like it does not compile, it will in fact *compile* and *run* without issue.
Note that the =semicolons= separating the three sections *are* required, as ~for( )~ *without* any semicolons will *not* compile.

#+BEGIN_SRC java
for( ; ; )
    System.out.println("Hello World");
#+END_SRC

** 2. Multiple Terms

=initialization block= *requires* values to be of the same type. (In below code block: ~y~ and ~z~ are both *only* allowed to be ~long~.)

This code demonstrates three variations of the ~for loop~ you may not have seen.
- First, you can *declare* a variable, such as ~x~ in this example, before the loop begins and use it after it completes.
- Second, your =initialization block=, =boolean expression=, and =update statements= can include extra variables that *may* or *may not* reference each other. For example, ~z~ is defined in the initialization block and is never used.
- Finally, the =update statement= can *modify* multiple variables.

#+BEGIN_SRC java
int x = 0;
for(long y = 0, z = 4; x < 5 && y < 10; x++, y++) {
    System.out.print(y + " "); }
System.out.print(x + " ");

#+END_SRC

** 3. Redeclaring a variable in the Initialization Block

*** Does not compile

#+BEGIN_SRC java
int x = 0;
for(int x = 4; x < 5; x++) // DOES NOT COMPILE
    System.out.print(x + " ");
#+END_SRC

We can fix this loop by removing the =declaration= of ~x~ from the *for loop* as follows:

*** Does compile

#+BEGIN_SRC java
int x = 0;
for(x = 0; x < 5; x++)
    System.out.print(x + " ");
#+END_SRC


** 4. Using incompatible Data Types in the Initialization Block

The variables in the =initialization block= *must* all be of the same type.
In this example, they ~y~ and ~z~ have different *types*, so the code will *not compile*.
#+BEGIN_SRC java
int x = 0;
for(long y = 0, int z = 4; x < 5; x++) // DOES NOT COMPILE
    System.out.print(y + " ");
#+END_SRC

** 5. Using Loop variables outside the loop
If you notice, ~x~ is defined in the =initialization block= of the loop and then used after the loop terminates.
Since ~x~ was *only* scoped for the loop, using it *outside* the loop will cause a *compiler error*.

#+BEGIN_SRC java
for(long y = 0, x = 4; x < 5 && y < 10; x++, y++)
    System.out.print(y + " ");
System.out.print(x); // DOES NOT COMPILE
#+END_SRC
