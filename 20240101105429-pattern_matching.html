1. Pattern Matching

  • Introduced in Java 16

Is a technique of controlling program flow that only executes a section of code that meets certain criteria. It is used in conjunction with if Statements statements for greater program control.

1.1. Legacy (pre 16)

Consider the following code that takes a Number instance and compares it with the value 5: 

void compareIntegers(Number number) {
    if(number instanceof Integer) {
        Integer data = (Integer)number;
        System.out.print(data.compareTo(5));
    }
}

The cast is needed since the compareTo() method is defined on Integer, but not on Number.

Code that first checks if a variable is of a particular type and then immediately casts it to that type is extremely common in the Java world. (See below for shorter variant)

1.2. Patter Variables (Java 16)

void compareIntegers(Number number) {
    if(number instanceof Integer data) {
        System.out.print(data.compareTo(5));
    }
}

The variable data in this example is referred to as the pattern variable. Notice that this code also avoids any potential ClassCastException because the cast operation is executed only if the implicit instanceof operator returns true.

1.2.1. Reassigning Pattern Variables

While possible, it is a bad practice to reassign a pattern variable since doing so can lead to ambiguity about what is and is not in scope: 

if(number instanceof Integer data) {
    data = 10;
}

The reassignment can be prevented with a final modifier, but it is better not to reassign the variable at all. 

if(number instanceof final Integer data) {
    data = 10; // DOES NOT COMPILE
}

1.3. Pattern Variables and Expressions

Pattern matching includes expressions that can be used to filter data out, such as in the following example: 

void printIntegersGreaterThan5(Number number) {
    if(number instanceof Integer data && data.compareTo(5) > 0)
        System.out.print(data);
}

We can apply a number of filters, or patterns, so that the if statement is executed only in specific circumstances. Notice that weâ€™re using the pattern variable in an expression in the same line in which it is declared.

1.4. Subtypes

The type of the pattern variable must be a subtype of the variable on the LHS of the expression. 

Integer value = 123;
if(value instanceof Integer) {}
if(value instanceof Integer data) {} // DOES NOT COMPILE

While the second line compiles, the last line does not compile because pattern matching requires that the pattern variable type Integer be a strict subtype of Integer.

1.5. Flow Scoping

The compiler applies flow scoping when working with pattern matching. Flow scoping means the variable is only in scope when the compiler can definitively determine its type.

1.5.1. Exam trick

It might surprise you to learn this code does compile: 

void printOnlyIntegers(Number number) {
    if (!(number instanceof Integer data))
        return;
    System.out.print(data.intValue());
}

The method returns if the input does not inherit Integer. This means that when the last line of the method is reached, the input must inherit Integer, and therefore data stays in scope even after the if Statement ends.
