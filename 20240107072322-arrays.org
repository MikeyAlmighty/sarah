* Arrays

An =array= is an *area of memory* on the [[id:dbcc90ea-bf9d-4bf5-9872-36b5794b88f0][Heap]] with space for a designated number of elements.

=Arrays= in Java are *objects*, and you can use them to store *multiple* *values*.

A [[id:afe76105-5f94-4dac-a929-e7a7dcdb5983][String]] is *implemented* as an ~array~ with some methods that you might want to use when dealing with characters specifically.

A [[id:ffe9bdf4-50cb-432a-85bc-5a32200380a4][StringBuilder]] is implemented as an ~array~ where the array object is *replaced with a new, bigger array object* when it runs out of space to store all the characters.

If we didn’t want to use a [[id:afe76105-5f94-4dac-a929-e7a7dcdb5983][string]] for some reason, we could use an array of char [[id:46582863-7f97-448d-bbc7-f7da4c1e7574][primitives]] directly:

#+BEGIN_SRC java
char[] letters;
#+END_SRC

(This wouldn’t be very convenient because we’d lose all the =special= properties String gives us)

** Syntax
#+BEGIN_SRC java
double[] myList = new double[10];
#+END_SRC

*** .toString()
Java provides a method that prints an array nicely - ~Arrays.toString(X)~:

#+BEGIN_SRC java
String[] names = { "DeadPool", "BatMan", "AquaMan" };
System.out.println(names.toString()); // [Ljava.lang.String;@3fee733d
System.out.println(Arrays.toString(names)); // [DeadPool, BatMan, AquaMan]
#+END_SRC
