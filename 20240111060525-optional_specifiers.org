* Optional Specifiers

Unlike with [[id:0d43fdee-5463-4f66-8b65-ae9a7b4b74e2][Access Modifiers]], you can have *multiple* specifiers in the same method (although not all combinations are legal).

When this happens, you can specify them in *any order*.

You can have *zero* or *more* ~specifiers~ in a ~method declaration~.

#+ATTR_ORG: :width 650
[[./assets/optional_method_specifiers.png]]

#+BEGIN_SRC java
public class Exercise {
    public void bike1() {}
    public final void bike2() {}
    public static final void bike3() {}
    public final static void bike4() {}
    public modifier void bike5() {} // DOES NOT COMPILE
    public void final bike6() {}  // DOES NOT COMPILE
    final public void bike7() {}
}
#+END_SRC

** _Exam Trick_

While [[id:0d43fdee-5463-4f66-8b65-ae9a7b4b74e2][Access Modifiers]] and [[id:78bf41e5-259a-4c97-8df7-1685989e3d8c][Optional Specifiers]] can appear in any order, they must all appear *before* the ~return~ type.

Remember, [[id:0d43fdee-5463-4f66-8b65-ae9a7b4b74e2][Access Modifiers]] and [[id:78bf41e5-259a-4c97-8df7-1685989e3d8c][Optional Specifiers]] can be listed in *any order*, but once the ~return~ type is specified, the rest of the parts of the method are written in a specific order: ~name~, ~parameter list~, ~exception list~, ~body~.
